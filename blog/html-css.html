What are the best practices associated with using classes vs. ids?

Oscar Harron July 1st, 2022

Nice one! You've set up your first website using HTML. Your excitement builds as you open your marvelous creation and... it looks like this.

Maybe I am not the Picasso of the Web you think to yourself. No worries. Lets use CSS to add some style to your webpage. This can be done using selectors.

In CSS, selectors are used to target an HTML element on your webpage. Once we have an element selected we can use different styles to change how our element looks. This could be by using the color property to add a dash of colour to your text or by using the font-size property to change its size.

There are many different types of selectors in CSS but the two that we are looking at today are Class and ID.

ID vs Class: Adding selectors to your HTML

We will start with ID. ID's are unique and should only be used once on a page. If more than one element requires a particular style then you should reconsider using an ID as a selector.

To give an element an ID, give it the attribute id="your-id-here"

See the Pen Untitled by Ocsar (@drenchoman) on CodePen.

In the above, I have given my div an ID of 'supreme-orange-box'. This ID should only be used once. No other element on my webpage should have the ID of "supreme-orange-box"

Classes on the other hand can be used more than once. Using a class means that you can set a style and use that style on as many elements as you would like.

To give an element a class, give it the attribute of class followed by your class name. For example class="awesome-paragraph". You can also add multiple classes to an element like I have done in the codepen example, giving my div the class of 'box' and 'blue'.

"
ID for one, Class for many.

Using Class in your CSS.

To use the Class selector we type . followed by your classname-here. Add styling using CSS properties such as color or font-size

An element can have multiple classes but it is important to note that classes can override eachother if a particular property is specificed twice.

In the Codepen below, my div has the class of 'box' and 'yellow'. Under the CSS section you can see that the 'box' class has the background-color: property set to yellow. Below this selector we have a class of 'blue' which has the background-color: property set to blue.

See the Pen Untitled by Ocsar (@drenchoman) on CodePen.

Since our 'blue' selector comes after the 'box' selector, our box becomes the colour blue overriding the background-color property we set in the box class.

Try moving the 'blue' class above the 'box' class and see what happens.

See the Pen Untitled by Ocsar (@drenchoman) on CodePen.

As you can see our box is now yellow! This highlights the importance of where our class selectors are placed in our CSS.

Using ID in your CSS

To use the ID selector we type # followed by your id-here and then add our styling under the ID selector using CSS declarations

What makes ID so unique is that it offers a higher level of specificty when compared to classes. This means that the ID selector will override any of the properties declared under a class selector, no matter where the ID selector is written in your style sheet.

Using the same Codepen as before we can see that our box also has an ID of 'orange'. Try uncommenting the orange ID and see what happens to the colour of our box. Is it what you expected?

See the Pen Untitled by Ocsar (@drenchoman) on CodePen.

Our box has turned orange despite being at the top of our stylesheet. The ID selector overrides the background-color property specified by our 'box' and 'blue' classes even though they appear further down in our stylesheet.

ID's can be pretty powerful eh?

Other features

One other cool feature of using an ID is that we can navigate to any element that uses an ID. This can be useful for navigation and accessibility.

Try clicking here and watch the page navigate to the 'Using ID in your CSS' header.

I recommend you use this feature on elements such as your Page header or Footer. These elements are often unique to a webpage and are appropriate uses of the ID selector.

Final thoughts

I hope you learnt the differences between ID and class and the best practices around using them. If you are ever wondering whether I should use a Class or ID, remember the simple rule.

"
ID for one, Class for many.
